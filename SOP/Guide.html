SpectralCurveTool · 使用指南

面向 4108 设备（2560 像素）NIR 光谱 CSV 的桌面对比/校准分析工具 · v1.1

导航： 概述 安装与启动 数据格式 功能详解 校准分析 典型工作流 项目结构 数学原理 排错

一、概述

本工具用于对 4108 近红外光谱仪导出的 CSV 进行可视化对比与校准/映射分析。主要解决：

  • 多曲线叠加对比：拖入多个 CSV/文件夹，叠加显示，逐条显隐、改色、删除。
  • 跨量纲混排：原始强度（~千级）与吸光度（~0.2）可在同一图上以双 Y 轴或归一化对比。
  • 差异量化：相减 / 比值(带下限) / 有界相对偏差 / RMSE 评分。
  • 校准传递：单组/批量拟合仿射映射 B = m·A + c，按可靠性加权期望求整组系数，并可套用到其它曲线验证。

主题采用 Professional Scientific 风格（海军蓝 + 翠绿 + Slate 灰），基于 tkinter + matplotlib。

二、环境与安装

依赖

tkinterdnd2 >= 0.5.0      # 原生拖拽（可选；未装则用按钮加载）
matplotlib  >= 3.10
pandas      >= 2.2
numpy       >= 2.1

安装：

pip install -r requirements.txt
说明：tkinter 随 Python 自带。tkinterdnd2 提供原生拖拽；若安装失败或缺失，程序仍可运行，只是改用「添加文件/文件夹」按钮加载数据。

启动

在项目根目录执行（任选其一）：

python run.pyw          # 推荐：pythonw 运行，无控制台黑窗
python -m src           # 等价入口（会带控制台）
双击 run.pyw            # Windows 直接双击
注意：请从项目根目录启动，否则 src 包无法被正确导入。

三、支持的数据格式

4108 导出的 CSV 头格式不统一，本工具采用「定位首个全数值行」的鲁棒策略自动解析，兼容以下变体：

情形示例处理
标准两列CWL,Intensity / Wavelength,Absorbance自动识别
带 BOM﻿Wavelength,AbsorbanceUTF-8-sig 去除 BOM
无名标题行首行 ,名称，真正表头在第二行跳过标题行
特殊列名Wavelength[nm] / Log(1/R) / White Reference按文本判类型
自动跳过的非光谱文件：coeffA.csv / coeffB.csv 在 4108demo导出 下是单行平铺系数数组（无波长列），会被自动识别并跳过（数据行数值列数 > 2 即判为非光谱），避免解析成 1 点的废曲线。

Y 类型自动判定：表头含 absorb/log(1/r) → 吸光度；含 intensity/reference/dark → 强度；无表头按数值范围（>50 强度，否则吸光度）。

文件夹加载：递归收集 *.csv，自动排除 tmp 目录。

四、功能详解

  • 拖拽 把 CSV 文件或文件夹拖到拖拽区（或整个窗口任意位置）。
  • 按钮 添加文件 / 添加文件夹（递归）/ 清空（删除全部已加载曲线）。

操作效果

行文字颜色 = 曲线颜色，便于与画布图例对应。

4.3 绘图选项

Y 轴模式

模式用途

4.4 差值分析（差值基准 + 差值方式）

先在「差值基准」选一条作基准曲线，再选「差值方式」。底部出现独立子图：

方式公式参考线说明
相减y − basey=0绝对差值
比值(带下限)y / max(|base|, 5%峰)y=1相对倍数；分母设下限防爆
有界相对偏差(y−base)/(|y|+|base|)y=0恒在 [-1,1]；y=base 处为 0；0/0 填 0
RMSE(原值)√mean((yₙ−baseₙ)²)—标量；原始吸光度空间；底部显示评分表（每条 vs 基准，按 RMSE 升序）
右侧「清空显示」按钮：隐藏画布全部曲线，但保留已加载列表（与左侧「清空」不同，后者删除全部数据）。

五、校准分析（左侧「校准分析」区，三个窗口）

核心思想：两条归一化后形状一致的曲线，在原始尺度上必为仿射关系 B = m·A + c（见数学原理）。m=增益（积分时间/光源/探测器/浓度），c=基线（暗电流/杂散光）。

5.1 映射 / 校准 A→B（单组）

  1. 选曲线 A、B（下拉框宽度自适应文件名）。
  2. 自动在 A、B 波长重叠区间最小二乘拟合 B ≈ m·A + c。
  3. 结果行给出 m、c、R²、Pearson r、残差RMS + 自动判断（R²>0.999 良好 / >0.95 部分吻合 / 否则勿用）。
  4. 三张子图：① 原始 A vs B；② 映射后 A′=mA+c vs B（应重合）；③ 残差 ε=B−A′。
  5. 「导出结果」→ CSV（含 m,c,R²,r + 逐波长表）。

5.2 批量映射（求平均/期望）

对多组 A→B 各自拟合，再聚合得到整组映射系数。

配对构建

  • 手动 源 A + 目标 B →「添加配对」。
  • 智能 智能配对 S1–S6：以当前所选源/目标参考曲线的文件夹 + 变体后缀分组，自动匹配 S1↔S1 … S6↔S6（排除 _absorbance/_raw 混配）。

聚合方式（关键）

方式公式特点
期望(残差倒数加权) 默认wᵢ=1/RMSᵢ²，E=Σwᵢxᵢ/Σwᵢ反方差加权=最大似然合并；残差越小权重越大，高偏差组被压低，不被离群带偏
平均(算术)mean等权，易被坏组拉偏
中位数median + 标准化MAD最抗离群

结果表给出每组 m/c/R²/r + 聚合行 + 离散度行；子图网格每组单独一张残差图（不叠加）。

5.3 批量校准验证（套用系数）

把已知 (m, c) 套到一组源曲线，与目标组逐配对对比，验证系数好坏。

系数来源（3 个按钮）

  • 使用上次 A→B 结果：单组拟合结果。
  • 使用批量映射平均：本会话批量映射的聚合系数。
  • 从 CSV 导入系数：读取「批量映射」或「映射 A→B」导出的 CSV（自动取其平均值/系数）。

输出（两个标签页）

  • 残差叠加：各配对残差叠一张图 + 逐配对 RMS/R²/r 表 + 平均/最大 RMS 汇总。
  • 还原曲线对比：在配对列表多选要看的曲线，显示 源 A（黑虚线）/ 校准 A′=mA+c（粗实线）/ 目标 B（粗实线）——校准好的话 A′ 与 B 重合。

5.4 导出

入口导出内容
主窗口「导出 PNG」当前画布图片
主窗口「导出汇总 CSV」公共波长网格 + 各可见曲线插值对齐
映射 A→B「导出结果」m,c,R²,r + 逐波长 A/B/A′/ε
批量映射「导出结果」聚合 m,c（含方式标注）+ 离散度 + 每组明细

六、典型工作流

流程 A：曲线叠加对比

拖入文件夹 → 曲线列表勾选关心的几条 → Y 轴模式选「双轴」或「归一化」→ 工具栏缩放查看 → 导出 PNG。

流程 B：求相似度

差值基准选一条 → 差值方式选 RMSE(原值) → 底部评分表按 RMSE 升序排列，看谁最像基准。

流程 C：校准传递（完整闭环）

  1. 批量映射（求平均）：智能配对 S1–S6 → 聚合方式选「期望」→ 得整组 m、c → 导出 CSV。
  2. 批量校准验证：「从 CSV 导入系数」读入上一步的平均值 → 计算残差 RMS。
  3. 切「还原曲线对比」页，多选曲线看 A′ 是否与 B 重合 → 判断系数是否可推广到整批。

八、数学原理速览

8.1 仿射映射（为什么归一化后一致 ⟺ 仿射关系）

min-max 归一化 n(y)=(y−min)/(max−min) 对仿射变换不变：
若 B = m·A + c（m>0），则 n(B) ≡ n(A)

故「归一化后一致、原始不同」⟺ A、B 为仿射关系。映射 A→B = 求 (m,c) = 线性回归/仪器校准。m=增益，c=基线。

8.2 加权期望（反方差合并）

E[x] = Σ(wᵢ·xᵢ) / Σ(wᵢ)， wᵢ = 1 / RMSᵢ²

把每组拟合当成方差 ∝ RMS² 的高斯测量做最大似然合并：残差小（可靠）的组权重大，残差大（偏差）的组权重趋零，因此不被离群带偏——这正是区别于等权算术平均的关键。

8.3 有界相对偏差

D = (y − base) / (|y| + |base|) ∈ [−1, 1]

y=base 时 D=0（一致）；两值同时为 0 的退化点 0/0 填 0（语义上等于一致），避免 NaN/inf 尖峰。

8.4 RMSE（原值）

RMSE = √( mean( (yₙ − baseₙ)² ) )， 0 = 完全一致，单位 = Absorbance

在原始吸光度空间计算：基准曲线插值到对方波长网格后逐点相减求均方根。越小越接近基准。

九、排错

现象原因 / 处理
ModuleNotFoundError: src未在项目根目录启动。请 cd 到根目录再 python run.pyw。
拖拽无反应tkinterdnd2 未装。pip install tkinterdnd2，或改用「添加文件」按钮。
图例出现方框（缺字符）个别符号在某些字体缺失；本工具已尽量用通用符号，可忽略。
曲线点数异常（如 1 点）该文件可能是单行系数数组（非光谱），会被自动跳过；若仍出现，检查 CSV 是否两列。
中文乱码（matplotlib）系统缺中文字体；app.py 已配置 Microsoft YaHei/SimHei，一般无需处理。
批量验证配对混乱请用「智能配对 S1–S6」或「添加配对」显式构建；切勿依赖下标。
